# 査読レポート（ALife Conference・Full Paper）

## 1. 概要（Reviewer summary）
本稿は、**最小自己複製体が「生命らしさ」を獲得していく過程**を、人工生命（ALife）世界の中で操作的に測定する枠組みとして **Semi-Life** を提案する。Viroid / Virus / ProtoOrganelle の3つのアーキタイプに対し、V0（複製）→V5（段階的ライフサイクル）までの **Capability Ladder** を段階的に付与し、資源ハーシュネス（4段階）を横軸に、
- 生存（step 500 の alive count）
- 複製回数
- 内部化度（Internalization Index; II = Eint/(Eint+Eext)）
などで、ウイルス様（II≈0）から生命様（II>0）への遷移を相図として示す。

また、H1–H7の**事前登録（pre-registration）仮説**を設定し、Holm–Bonferroni補正を含む統計手続きを明示し、環境依存の「コスト谷（cost valley）」—特にV1/V2でのコスト増と、V3代謝追加による急回復—を主要な結論として主張している。

## 2. 主要貢献（Contribution & novelty）
1. **「ウイルス→生命」遷移を能力の段階的内部化として操作定義**し、相図として提示する設計は分かりやすい。
2. **事前登録 + 多重検定補正**を前面に出した実験計画は、ALife分野でも実務的価値が高い。
3. V4/V5で見える**環境依存の反転（rich で不利、scarce で有利等）**は、単調な「能力が増えるほど良い」モデルへの反証として面白い。

## 3. 強み（Strengths）
1. **問題設定が明確**：境界的存在（viroid/virus/proto-organelle）を連続体として捉える方向性は説得力がある。
2. **設計が実験として読みやすい**：能力をbitmaskで追加し、各能力のコストを明示している。
3. **統計プロトコルが丁寧**：検定手法、補正、探索的解析の区別が明確。
4. **可視化（相図）の説得力**：capability×harshnessの地図は読者の理解を助ける。

## 4. 重大な懸念（Major concerns / reasons to reject unless addressed）

### M1. V1（Boundary）の「利益」がモデル上で弱く、結果がほぼ自明に見える
V1は **(i) 毎stepの修復コスト**を追加し、同時に **(ii) integrity 低下で死亡**という新たな致死機構を導入する。一方で、V0-onlyに対してV1が提供する「保護的利益」が、現行記述だと**明確に定義されていない**（少なくとも本文に現れない）。
- その結果、H1が全環境でδ=1.00になるのは、**V1がコストと制約（死亡条件・複製ブロック）だけを増やしている**ために見え、キャップシド類似としての解釈（「保護のための境界」）が弱い。

**改善提案**：
- 境界が守るべき外的ハザード（拡散、資源リーク、損傷イベント、捕食、ノイズ注入など）を明示し、V1がそれを軽減する形で利益を入れる。
- もしくは、V1を「単なるコスト」ではなく、例えば **内部プール保持（V3）・反応（V4）**の前提条件として必須化する等、能力間依存を設計し直す。
- さらに、**boundary_decay_rate/repair_cost の感度解析**（sweep）で、V1が有利になる領域が存在することを示す。

### M2. Internalization Index（II）が実質的に「V3のオン/オフ」指標に近い
IIの定義上、V0–V2はII=0で、V3追加で急に0.4–0.6台へ上がる。これは綺麗ではあるが、
- 「ウイルス様→生命様の連続軸」という主張に対し、IIが**V3の存在にほぼ支配**され、V4/V5では変化しない。
- 結果として、**生命らしさの多面性（制御・応答・ライフサイクル等）をIIが表現していない**。

**改善提案**：
- IIを「内部化」の一般指標に拡張（例：境界維持に必要な資源の内部調達比率、センサー行動で得られるエネルギー寄与など、複数チャネルの内部化率を合成）し、V4/V5でも変化しうる形にする。
- あるいは、IIは「代謝内部化」に限定した指標として位置づけ直し、**別の連続指標**（例：自己維持の情報量、制御の安定性、ライフサイクルの規則性など）を併記する。

### M3. 効果量が飽和（δ=1.00が多い）し、一般性・妥当性より「設定依存」を疑わせる
多数の比較でδ=1.00が出るのは、ロバストである可能性もある一方、
- 初期個体数が10で、500 stepという設定では**確率過程の幅が狭く、境界条件で全滅/爆発が起きやすい**。
- 事前のパラメータキャリブレーション（seed 0–49）による「最も見やすい領域」を選んだ可能性を完全に排除できない（悪意ではなく、研究設計上の自然なバイアス）。

**改善提案**：
- ninit（初期個体数）とrun lengthを伸ばした再現（例：ninit=50、T=2000）を少数条件でも追加。
- 各能力の主要パラメータ（replication_cost, boundary cost, move cost, conversion rate）の**部分的sweep**で、結論が位相的に保たれるかを示す。

### M4. 「背景プラットフォーム（七基準世界）」の必要性がやや弱い
Semi-Lifeは既存の七基準ALife世界に置かれるが、本文では
- 生物（organism）と直接相互作用しない
- 共有資源場から吸うのみ
とされるため、「七基準世界に置くこと」が、単なる資源場単独の実験に比べてどの点で本質かが見えにくい。
また、共有資源なら本来は他個体へ影響しうるため、
- 「organism population is not affected」という記述の整合性（吸う量が微小なのか、資源場が別なのか）が不明。

**改善提案**：
- 背景世界由来の「資源景観の構造」「時空間的変動」「競合圧」など、ここでしか得られない要素を明示。
- 共有資源が他者に影響しない設計なら、その実装（別レイヤー資源、固定供給、等）を明確化。

## 5. 追加で気になった点（Minor comments）
1. **死亡条件の全体像**：V0-onlyは何で死ぬのか（エネルギー枯渇で死ぬのか等）を明確に。
2. V4の初期重みがchemotaxisに偏る（w1=w2=0.5）点は「進化の結果」ではなく「初期バイアス」なので、結果解釈の脚注を厚く。
3. H7の「全体単調」主張は、JTで有意でも、ペアワイズ反転がある以上、結論の書き方は慎重に（“global trend dominated by V3” など）。
4. 図表：Figure 3/4の読み取りを補う、定量的要約（相関係数、回帰の傾き等）があると強い。

## 6. 著者への質問（Questions for authors）
1. V1の「保護利益」は何に対するものですか？現行モデルだとコスト・制約増に見えるため、利益の実装とその根拠を明確にしてください。
2. 「背景七基準世界」を使う必然性は何でしょうか？資源場だけのモデルとの差分を一言で説明できますか？
3. IIを「生命らしさの連続軸」と呼ぶ際、V4/V5のような情報処理・行動能力をどのように位置づけますか？
4. キャリブレーション（seed 0–49）で調整したパラメータの目的関数（何を良くしたか）を、再現可能な形で説明できますか？

## 7. 改善提案（Actionable suggestions）
1. **V1の利益を導入**し、V1が有利になる領域（位相）を示す（感度解析込み）。
2. IIの拡張 or 再位置づけ：代謝内部化の指標として限定し、別の連続指標を追加。
3. 追加実験：ninitとTを増やした追試（主要条件だけで良い）。
4. 背景世界の意義：資源景観の違い、他者影響の扱いを明確化。

## 8. 総合評価（Recommendation）
- **総合判定**：**Weak Reject（改稿で化ける可能性が高い）**
- **主理由**：V1の利益設計が弱く、主要結論（cost valley）の一部がモデル定義の自明さに見える点。IIの連続軸主張も現状ではV3依存が強い。
- ただし、事前登録・相図・環境依存反転の観察は価値が高く、上記M1/M2が解消されれば **Weak Accept以上**に上がり得る。
