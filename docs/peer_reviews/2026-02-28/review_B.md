# ALIFE 2026 Full Paper Review: "Semi-Life: A Capability Ladder for the Virus-to-Life Transition"

## 要約
本稿は、最小複製子が「生命らしさ」を獲得していく過程を、**Capability Ladder (V0–V5)** と **Internalization Index (II)** により定量化するALife実験系を提示する。Viroid / Virus / ProtoOrganelle の3つの原型を、共通の7基準ALifeワールド上で比較し、能力追加が生存に与える影響を事前登録仮説（H1–H7）で検証する。主要結果は、V1/V2が生存を低下させる「コスト谷」→V3で大幅回復、V4/V5は環境依存の利得/損失を持つという非単調性である。統計的には Holm-Bonferroni 補正、Cliff’s δ とJT検定を用い、効果は非常に大きい。

## 強み
1. **明確な操作化**：能力を明示的なプロセスとコストとして定義し、段階的に付加する設計が理解しやすい。
2. **事前登録と多重比較補正**：仮説・統計法の透明性が高く、探索的分析と区別されている。
3. **環境軸の導入**：資源ハーシュネスを4段階で検証し、能力効果の環境依存性を示している点はALife的に重要。
4. **結果の語り口が明快**：V1/V2のコスト谷やV4/V5の反転効果など、読者が直感的に理解しやすい。

## 主要な懸念（Major Concerns）
### 1) V1（境界維持）の「利得」がモデルに明示されていない
V1は「境界が崩れると死亡／複製不能」「修復コストがかかる」と記述されている一方で、**境界があることによる保護効果や外的ハザードが明示されない**。現状の定義ではV1は純粋なコストになり、H1の「生存低下」は構造的に当然の結果になっている可能性が高い。**V1が得る利点（例：資源漏洩の抑制、外的ダメージの遮断、内部プールの保持）を明確化し、モデル内に実装する必要**がある。

### 2) V2（ホームオスタシス）も同様に「機能的利点」が不十分
V2は取り込み率のスロットリングを行うが、取り込み行動自体のコストや副作用が不明なため、利得がほぼ発生しない設計に見える。結果としてV2が無効であることは想定通りになる。**「制御しないことのデメリット」を明示し、V2の機能的妥当性を示す必要**がある。

### 3) 結果の大半が“設計パラメータ依存”である可能性
効果サイズが多数の条件で δ=1.00 に張り付いており、結果が**極端に分離**している。著者も認識しているが、**特定パラメータ設定による脆弱な結論**になっている恐れがある。V1–V5のコスト/しきい値/変換率について**感度分析（少なくとも2–3点のパラメータスイープ）**が不可欠。

### 4) Internalization Index (II) が「生命らしさ指標」として単調でない
IIは定義上 V3 の内部変換率にほぼ直接支配され、V0–V2は常に0。**連続的な「生命らしさ軸」という主張に比べ、離散的かつ設計依存の指標**に見える。IIが生存や持続性と独立な情報を持つことを示すには、**IIの時間平均・分布・他指標との相関/非相関の検証**が必要。

### 5) 「環境が共有資源を使うが影響はない」という記述の矛盾
本文で「SemiLife は同一資源場から取り込むが、背景生物は影響を受けない」とある。**資源共有であれば相互影響が生じるはずで、この点の記述は矛盾**している。もし資源場が実際には独立なら、その旨を明示すべき。

## 追加で確認したい点（質問）

2. V4 の線形ポリシーは進化によりどの程度変化しているか？ 初期重みが結果を支配していないか？
3. 500ステップは定常状態に十分か？ 長期（>1000 steps）でパターンが維持されるか？
4. 結果の再現性：乱数シード以外の要素（例：初期位置配置、資源初期配置）がどこまで固定・ランダムか？

## 軽微なコメント（Minor Comments）
- H4/H7 にJT検定を用いることは理解できるが、非単調な系列に対しては解釈の注意が必要（「単調増加」を示したとは言いにくい）。
- H3（ProtoOrganelle解放）は、ベースラインが複製不能であるため差が自明であり、検証価値が低い。もう少し意味のある比較設定が望ましい。
- 相図や図中で平均値のみならず分散（エラーバー、箱ひげ、CI）を併記すると説得力が上がる。

## 総合評価
この研究は、ALifeにおける「生命らしさの段階的獲得」を実験的に捉える意欲的な試みであり、事前登録・多重比較補正・環境依存性の分析など、方法論的な筋は良い。一方で、V1/V2の機能的利点がモデル内に実装されていない（あるいは記述されていない）ため、中心的結果が**設計上の必然**に見える点が最大の問題である。主要結論を主張するには、**V1/V2の利点の明示化と感度分析**が不可欠。

**Recommendation:** Weak Reject（大幅修正を前提とする）
